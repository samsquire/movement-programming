Movement programming

This is not finished and this is a prototype.

This is the beginnings of an experimental programming vision. It's based on the idea that we move things between places and build up programs by moving things around, between variables, locations, between functions and between method calls or APIs and generate loops and instructions based on the movement. The idea is that complicated data structures can be transformed and manipulated by the primitive of moving around.

The dream is of this idea is that algorithms such as btrees and quicksort shall be creatable with point and click.

If you're familiar with spreadsheets, you know that each cell can contain a formula or a value. This is similar except you can move things between locations and the movement creates the code or instruction.

  1. Click a cell
  2. Click where to move it to.
To create loops (incomplete):
  1. Click Loop
  2. Click source cell
  3. Click end of source cell
  4. Click destination cell

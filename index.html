Movement programming: Point and click

The dream is of this idea is that algorithms such as btrees and quicksort shall be creatable with point and click.

This is not finished and this is a prototype.

This is the beginnings of an experimental programming vision inspired by spreadsheets and assembly programming. It's based on the idea that programming is just logistics: we select then move the right things between places, so they're in the right place ready for the next instruction that does something based on that placement.

In a given context, we need to move the right things somewhere else to get the right chain of events to cause computation.

Such a large part of programming is deciding what needs to be used and doing the right thing with it.

Perhaps we move between variables, locations, between functions and between method calls or APIs and generate loops and instructions based on the movement. The idea is that complicated and deep data structures can be transformed and manipulated by the primitive of moving around the right things.

If you're familiar with spreadsheets, you know that each cell can contain a formula or a value. This is similar except you can move things between locations and the movement creates the code or instruction.

  1. Click a cell
  2. Click where to move it to.
To create loops (incomplete):
  1. Click Loop
  2. Click source cell
  3. Click end of source cell
  4. Click destination cell
